Project Motivation
The Problem: Fragmented Travel Systems

Travelers currently struggle with disconnected logistics:

Data Silos: Flights, Hotels, and Tours exist in separate systems, making planning impossible.
Manual Processing: Agencies manually calculate platform fees (5%) and taxes, leading to massive financial errors.
Inventory Lag: Without automation, seat availability is never up-to-date, causing overbooking issues.
Security Risks: Personal traveler details and payment records are often stored in insecure, open formats.
Why SmartGo?

SmartGo provides a Single Point of Truth. By treating every travel entity as an Object, we ensure that a booking automatically updates inventory and calculates the exact bill instantly. This removes human error and provides a seamless traveler journey.

SOLID Principles
Liskov Substitution (Predictability)

The Rule: A child class (User) must be 100% compatible with its parent (Person). If you expect a Person, a User should never break that expectation.

In SmartGo, we treat every account as a 'Person' during data loading. Because we follow LSP, the app never crashes when switching between Users and Admins.

// LSP in our Loading Logic
List<Person> members = DataStore.loadUsers();

for(Person p : members) {
  // We treat everyone as 'Person' safely
  System.out.println(p.toString());
}

Why this over Type Casting? Checking instanceof User every time is slow and prone to errors. LSP makes our code generic and stable.

OOP Topic 7 (A)
Object Serialization (Pipe Format)
// From DataStore.saveUser()
public static void saveUser(User user) {
  String line = user.getId() + "|" +
    user.getName() + "|" +
    user.getEmail() + "|" + ...;
  appendLine(USERS_FILE, line);
}
// Why Pipe Delimiters?
// 1|Abdul Rehman|rehman@email.com|...

// This allows for infinite fields
// without confusing the parser.
Comma Separated (CSV): Comma-based files break if a user types a comma in their address (e.g., "Street 1, Apt 2").
VS
Pipe Delimited: The pipe symbol (|) is rarely used in text, making our data storage 100% reliable for addresses and comments.
